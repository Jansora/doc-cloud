# 环境准备
> 如果是本地私有部署, 在资源紧张的情况下, 可通过 PVE 虚拟化出三台 `Linux` 机器 (可视情况进行内存超配)


三台主机 (一台 Master, 两台 Worker). 推荐 Master 节点为 2C4G. Master 节点配置可更高
要求:
1. 系统为 `Ubuntu 22.04`
2. 内网互通
3. 所有节点都需要用 `Root` 用户操作
> 非 Root 未验证

待安装的 Docker: 19.03.15
待安装的 Kubernetes: 1.19.16
> kubernetes 从 1.20 开始 docker 不作为默认运行时了

目前三台节点的配置如下.

| 角色 | 主机名 |  网络   |
| ---- | ---- | ------- |
|  Master     |  U1  |   `inet 192.168.88.11  netmask 255.255.255.0  broadcast 192.168.88.255` |
|  Worker 1   |  U2  |   `inet 192.168.88.12  netmask 255.255.255.0  broadcast 192.168.88.255`   |
|  Worker 2   |  U3  |   `inet 192.168.88.13  netmask 255.255.255.0  broadcast 192.168.88.255`   |


## 配置 NFS 共享存储

1. 在U1 上安装 NFS 服务器
2. 在 U2 / U3 上挂载 NFS 共享目录

见 [配置 NFS 共享存储](/cloud-native/install-k8s/install-nfs)
